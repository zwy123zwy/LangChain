# 官方生态与选型

本项目**完全采用 LangChain / LangGraph 官方 JavaScript 生态**，所用能力均来自官方 npm 包与推荐用法。

---

## 一、本项目使用的官方包

| 包名 | 用途 | 官方文档 |
|------|------|----------|
| **@langchain/core** | 消息、Prompt、LCEL、Runnable、OutputParser 等核心抽象 | [js.langchain.com](https://js.langchain.com/docs/) |
| **@langchain/ollama** | 对接本地 Ollama（ChatOllama、OllamaEmbeddings） | [@langchain/ollama](https://js.langchain.com/docs/integrations/chat/ollama) |
| **@langchain/langgraph** | 图编排、createReactAgent 等预构建 Agent | [LangGraph JS](https://langchain-ai.github.io/langgraphjs/) |
| **@langchain/langgraph-sdk** | 调用远程 LangGraph 服务（Platform / 自建） | [LangGraph SDK](https://github.com/langchain-ai/langgraph-sdk-js) |
| **@langchain/community** | 文档加载、向量库、Tavily 等社区集成 | [Integrations](https://js.langchain.com/docs/integrations/) |
| **@langchain/textsplitters** | 文本拆分（RecursiveCharacterTextSplitter 等） | [Text Splitters](https://js.langchain.com/docs/modules/data_connection/document_transformers/) |

所有 API 与用法以 **js.langchain.com**、**LangGraph 官方文档** 为准。

## 三、官方生态中的后端/服务化支持

**有。** 官方在 JavaScript 生态里通过 **LangGraph** 提供后端服务能力，而不是单独的“LangServe for Node”：

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **LangGraph CLI** | 使用 `@langchain/langgraph-cli`，执行 `npx @langchain/langgraph-cli dev` 启动本地 API 服务（默认端口 2024），带 Studio 调试界面与 API 文档 | 本地开发、联调 |
| **LangGraph Platform** | 将 LangGraph 应用部署到官方托管，获得生产级 HTTP API、扩缩容与监控 | 生产部署 |
| **自建 Express** | 在本项目中用 Express 封装聊天、Agent、RAG 等接口 | 与现有业务同进程、自定义路由与鉴权 |

### LangGraph CLI 用法概要

1. 安装 CLI：`npm install -D @langchain/langgraph-cli`
2. 在项目中添加 `langgraph.json`，配置图入口，例如：
   ```json
   {
     "graphs": {
       "assistant": "./src/graph.js:graph"
     },
     "node_version": "20"
   }
   ```
3. 启动本地服务：`npx langgraph dev`（或 `npx @langchain/langgraph-cli dev`）
4. 本地会提供 HTTP API 与 LangGraph Studio；前端或其它服务用 `@langchain/langgraph-sdk` 调用。

**本仓库中的官方后端**：已提供 **`langgraph-server/`** 目录，按官方结构配置，可用 CLI 启动：

```bash
cd langgraph-server && npm install && npx @langchain/langgraph-cli dev
```

或从仓库根目录执行：`npm run langgraph:dev`。详见 [langgraph-server/README.md](../langgraph-server/README.md)。

**结论**：若希望**完全采用官方后端**，使用 **`langgraph-server/` + LangGraph CLI（本地）/ LangGraph Platform（云端）**；根目录的 Express（`server.js`）为**自建**，可与官方服务并存或通过 SDK 调用 `localhost:2024`。

## 五、可选扩展（仍为官方生态）

- **更多模型**：`@langchain/openai`、`@langchain/anthropic`、`@langchain/google-genai` 等，见 [Integrations](https://js.langchain.com/docs/integrations/)  
- **向量库**：通过 `@langchain/community` 使用 Chroma、Pinecone、Weaviate 等  
- **LangGraph Platform**：将图部署到云端，本项目用 `@langchain/langgraph-sdk` 调用  
- **LangSmith**：可观测与评估，需在 [LangSmith](https://smith.langchain.com) 配置并设置环境变量  

以上均为 LangChain/LangGraph 官方或官方维护的生态，本项目在选型上**适用官方生态**，便于与文档和后续版本对齐。
