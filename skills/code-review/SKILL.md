# 代码审查技能

## 概述

本技能用于对用户提供的代码片段进行审查，侧重正确性、可读性、安全与最佳实践。

## 执行步骤

1. **理解代码**：确认语言与功能意图。
2. **检查正确性**：边界条件、空值、类型是否处理妥当。
3. **检查安全**：是否存在注入、敏感信息泄露等风险。
4. **检查可读性**：命名、结构、注释是否清晰。
5. **给出建议**：按「必须修改 / 建议改进 / 可选优化」分级输出。

## 输出格式

- 🔴 **必须修改**：影响正确性或安全的项
- 🟡 **建议改进**：可读性、性能或最佳实践
- 🟢 **可选**：锦上添花

## 示例

用户提供：`function add(a,b) { return a+b }`

可指出：缺少类型校验（若为数字相加建议先校验）、无 JSDoc、可考虑更清晰的命名。
